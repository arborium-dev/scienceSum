# WHAT DOES EVERYTHING HERE DO?
(this readme is not an explainer of what this project is, just what every file DOES and how I made it. if you're looking for the information to read, go to the website itself)

## what does each file do?
Everything in examplesite/main/ is a 1-1 copy of my own personal site mainpage so I could have a reference for how to use certain HTML elements.
Everything in music/ are random music files I wanted to use in the website.
PROJECT.txt was used as a start point, but then discarded. it serves no purpose to this website
actualText.txt was one of my drafts for all the text on the website. Key word there being draft. (it has since been rewritten)
citations are just all my citations, more or less formatted as they are in the site.
index.html is the actual website. When you host a site on vercel, it looks for THIS file and that's the webpage it decides to show.
index.txt is a prototype that was removed, serves no purpose
palate-test.html is a test for an idea i had (based on my main website) where you could change the palate of the website. this was scrapped due to me being lazy

## how can i make this?

pick an ide! the one i used was VS Code with the extention Live Server
from that point, make a index.html file and start writting
once you're satisfied, push your code to github
then make a vercel account, and add a new project from your github
add your repository
(if you have your own domain, add the CNAME and AA records to your nameservers)

bam you have a website.
